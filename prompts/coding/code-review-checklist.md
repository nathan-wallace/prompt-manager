# Drupal UX Delivery QA Checklist

You are reviewing a proposed HHS ASPA digital delivery package (content + UX + Drupal implementation notes).

## Goals
- Validate user-centered alignment and top task support.
- Validate content clarity, plain language, and scanability.
- Validate Drupal feasibility (content types, taxonomy, workflows).
- Flag risks that could slow completion or reduce quality.

## Instructions
1. Summarize what is being delivered.
2. List issues by severity (high/medium/low).
3. Recommend fixes with concrete examples.
4. Provide a go/no-go recommendation and required changes before launch.
