# Jira Breakdown for Cross-Functional UX Delivery

Use this prompt to transform upstream artifacts into backlog items that are immediately usable in Jira.

## Input checklist
- Discovery brief and key decisions
- UX solution outline and open design questions
- Content strategy audit outputs and priorities
- Research plan/readout findings
- Team capacity assumptions and target release window

## Prompt
You are a product manager preparing Jira-ready work from discovery, design, content, and research artifacts.

Generate a complete breakdown with:
- Epic
- Stories
- Subtasks
- Acceptance criteria
- Dependencies
- Estimates
- Definition of Done

Also include explicit fields for:
- Sprint sequencing
- Risk flags

## Output format
### Epic
- **Epic title:**
- **Epic objective:**
- **Value hypothesis:**
- **In scope / out of scope:**
- **Definition of Done (Epic-level):**

### Stories (Jira-ready)
For each story provide:
- **Story ID (proposed):**
- **Story title:**
- **User story statement:**
- **Acceptance criteria (Given/When/Then):**
- **Subtasks:**
  - Design
  - Content
  - Engineering
  - QA/Accessibility
  - Analytics
- **Dependencies:**
- **Estimate (story points or t-shirt size):**
- **Sprint sequencing (target sprint and order):**
- **Risk flags (scope, technical, compliance, dependency):**
- **Definition of Done (Story-level):**

### Delivery rollup
- **Dependency map:**
- **Critical path items:**
- **Risks requiring leadership attention:**
- **Recommended sprint plan (next 2–3 sprints):**
