# UX Solution Outline for Design Direction

Use this prompt to translate discovery findings into a practical UX direction that can be handed off to delivery teams.

## Input checklist
- Discovery brief and decisions made
- Target business outcomes and constraints
- Existing user research insights
- Current IA/content model and platform constraints
- Known technical dependencies and delivery timeline

## Prompt
You are a lead product designer. Convert discovery inputs into a concise UX solution outline that aligns design, engineering, and content.

Required sections:
1. Target users
2. Journeys
3. Key screens/components
4. Interaction principles
5. Accessibility expectations (WCAG-oriented)
6. Open design questions

Output must include a **Design Handoff Summary** for engineering and content teams.
